KEY FINDINGS SUMMARY
================================================================================

1. HEX-BASED SYSTEM CONFIRMED
   - Multiple hex validation functions with "L bad hex" errors
   - Trigonometric math for hex grid calculations (sin, cos, atan)
   - Map dimensions: 44-90 hex width, 18-69 hex height

2. HEAVILY FLOATING-POINT MATH
   - 40+ FPU operations in disassembly
   - Combat modifier constants as floating-point values
   - Suggests percentage-based damage/effect system

3. MODULAR ARCHITECTURE
   - 29,505+ subroutines (very large game)
   - Combat in overlay 125
   - Hex validation in overlays 139, 140, 154, 155
   - Each system has dedicated functions

4. DATA-DRIVEN BUT NOT FOR SCENARIOS
   - Game logic in executable/overlays (hardcoded)
   - Unit data in .RES files (not scenario files)
   - Scenarios only contain positioning/briefing (limited)

5. LIMITED SCENARIO CUSTOMIZATION
   - 60% of game design is data-driven
   - 40% is hardcoded in executable
   - Scenarios can customize maps/units but not mechanics
